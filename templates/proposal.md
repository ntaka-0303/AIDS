# **6. AIWorkforce（AIエージェントプラットフォーム）の概要**

## 6.1. 全体構成

* **業務自動化エンジン**
  - ノーコードで定義可能な **AIワークフローエンジン**
  - 営業・法務・バックオフィスなど複数部門を横断する共通基盤
* **AIエージェント基盤**
  - ユーザーの自然言語指示を理解し、最適なワークフローを自動選択・実行するエージェント
  - 不足情報のヒアリング、再生成、修正反映までを対話的に実行
* **ナレッジポータル**
  - 社内に散在する提案書、レポート、社内文書を一元的に蓄積・検索
  - キーワード検索＋ベクター検索による高精度な類似資料検索
* **文書処理機能**
  - PDF・Word・Excel・PowerPoint等の読解・要約・生成・比較・抽出・マスキング
  - 文書を構造化データとして再利用可能にする仕組み
* **インテグレーション**
  - Officeファイル、社内ポータル、基幹システム、ストレージ等との連携
  - 将来的な API 連携・Teams/Slack からのトリガー実行も見据えた設計

## 6.2. 特徴

* **営業・企画・DX推進部・情報システム部などが共通で利用できる横断型プラットフォーム**
* **ノーコード／ノープロンプト**：現場部門がプロンプト設計スキルに依存せずに利用・拡張可能
* **連続した業務フロー対応**：「調査 → 作成 → チェック → 登録」まで一貫して自動化
* **ナレッジとワークフローの連携**：過去提案書・各種レポートを参照しながら新規提案を自動生成
* **エンタープライズ向けのセキュリティ・運用基盤**

# **11. リスクと対策**

| リスク               | 内容                                      | 対策                                                         |
| ------------------ | --------------------------------------- | ---------------------------------------------------------- |
| AI出力の誤り・幻覚        | 提案内容・説明文に誤りが含まれるリスク                    | 根拠提示型RAG、レビュー・承認フロー、ガードレール設定                    |
| データ漏洩・権限逸脱アクセス   | 誤設定・誤操作による情報漏洩                         | RBAC、監査ログ、テナント分離、最小権限設計                              |
| 業務適合性・現場負荷       | 過度な自動化／現場業務との乖離                           | PoC段階での業務フロー設計、現場巻き込み、段階的なスコープ拡大                    |
| システム連携の複雑性        | レガシーシステムとの接続難易度                           | 段階的な連携範囲設定、API連携の標準化、DX推進部・情シスとの共同設計              |
| 組織・運用面での定着        | 担当者変更・異動による運用ノウハウの断絶                   | ナレッジ・運用プロセスの文書化、トレーニング、AIWorkforce上での運用手順テンプレート化 |
